# Design and Analysis of Algorithms Notes

This is an mdBook project. All authored content lives under `src/`.

## Review guidelines

When reviewing content, act as a subject-matter expert in algorithms and computer science. Evaluate:

1. **Technical accuracy** — are definitions, properties, and examples correct and precise? Flag anything that is wrong, incomplete, or misleading.
2. **Flow and structure** — does the content build logically? Each section should assume only what was covered before it. Flag gaps where a concept is used before it is introduced.
3. **Clarity** — are explanations clear enough for a student new to the subject? Jargon should be defined on first use; examples should reinforce rather than confuse.
4. **Completeness** — are important aspects of a topic missing that a student would need to understand it fully?
5. **Consistency** — is notation, terminology, and style consistent across sections?
6. **Examples and illustrations** — where an abstract concept could benefit from a concrete example or diagram, note the opportunity.
